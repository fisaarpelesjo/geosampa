# Instruções do Projeto

- Todo texto, mensagem ou resposta para copiar e enviar a outra pessoa deve ser entregue somente como texto final corrido, sem Markdown, sem listas e sem quebras de linha desnecessárias, salvo pedido explícito de outra formatação.
- Todo incremento implementado deve ser validado, commitado com título e descrição em português do Brasil e enviado para o remoto.
- Não versionar dados sensíveis, dados gerados, especificações locais do caso, endereços, SQLs específicos de investigação ou arquivos em `data/`.
- Evitar comandos `python -c` multiline com `\n` escapado no PowerShell. Para análises locais, preferir comandos PowerShell nativos, Python de uma linha simples ou outro método controlado que não dependa de escape frágil.
